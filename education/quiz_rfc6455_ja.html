References (URLs)

  • RFC 6455: The WebSocket Protocol

Goal: handshakeとOrigin checkを理解し, security boundaryを設計できるようにする.

Q1: WebSocket接続は通常どの形で始まる

Multiple Choice
Explanation: Correct (C): UpgradeとConnection headerを含むHTTP handshakeで開始する

Q2: WebSocketとTLS上のWebSocketに対応するschemeの組はどれ

Multiple Choice
Explanation: Correct (A): wsはplaintext, wssはTLS上

Q3: browserからvictimのcookieを使ってprivilegedなWebSocketを開かれるのを避けたい. 関連するcheckはどれ

Multiple Choice
Explanation: Correct (B): browserはhandshakeにOriginを送る. server側で検証するのが一般的

Q4: Sec-WebSocket-Accept計算の元になるrandom valueをclientが送るheaderはどれ

Multiple Choice
Explanation: Correct (A): clientがKeyを送り, serverがAcceptを返す

Q5: WebSocketについて正しい説明はどれ (複数選択)

Multi-Select
Explanation: Correct (A,B,C): framingとupgrade handshakeが要点 Options: - D (incorrect): 認証はappとdeployment次第

Q6: browser origin制御でよく使うrequest header名は何 (1語)

Short Text
Explanation: Correct: Origin
