  • RFC 5077: Session Tickets and TLS Extensions

Goal: ticketとは何か, 誰が守るのか, key漏えいの影響を理解する.

Q1: TLS session ticketの主目的として近いのはどれ

Multiple Choice
Explanation: Correct (C): serverが暗号化したblobをclientへ渡し, 後で提示させることでstateを外出しする

Q2: ticket内容のconfidentialityとintegrityを守る主体として正しいのはどれ

Multiple Choice
Explanation: Correct (B): ticket発行者はserverなので, server側のkey管理が中核

Q3: ticket keyを長期間使い回すときの大きなリスクとして近いのはどれ

Multiple Choice
Explanation: Correct (A): resume pathの設計や設定次第でforward secrecy目標が崩れ得る Related: key lifetimeとrotateが重要

Q4: session ticket運用の良い衛生として正しいものはどれ (複数選択)

Multi-Select
Explanation: Correct (A,B,C): ticketはperformanceだけでなくkey管理の問題 Options: - D (incorrect): ticket keyはserver secret

Q5: server side session cacheとticketの違いとして近いのはどれ

Multiple Choice
Explanation: Correct (B): cacheはstate storageが必要, ticketはclient側保持とserver key管理が必要

Q6: TLS sessionをresumeするためにclientが保存する暗号化blobを何と呼ぶ

Short Text
Explanation: Correct: ticket or session ticket
