References (URLs)

  • RFC 2818: HTTP Over TLS

Goal: https URLから, portとTLS前提の期待値を導けるようにする.

Q1: schemeがhttpsのURLが意味することとして最も近いのはどれ

Multiple Choice
Explanation: Correct (B): HTTPSはHTTP over TLSで, 通常はcertificateに基づくserver認証を行う Options: - A (incorrect): 同一hostがhttpとhttpsを提供するのは普通 - C (incorrect): Base64はsecurityではない

Q2: https URLのdefault portはどれ

Multiple Choice
Explanation: Correct (A): port指定がないhttpsは443へ対応付く

Q3: TLSがHTTP messageに対して提供する性質として正しいものはどれ (複数選択)

Multi-Select
Explanation: Correct (A,B,C): TLSはchannelを守るが, 誰が何を許可されるかは決めない Options: - D (incorrect): authorizationはapp側の責務

Q4: user inputからabsolute URLを組み立てる. credentialやtokenが関わるなら有効な対策はどれ

Multiple Choice
Explanation: Correct (C): security sensitiveなflowはhttps強制が基本

Q5: https schemeについて正しい説明はどれ

Multiple Choice
Explanation: Correct (A): scheme差はoriginやdefault portの差につながる

Q6: httpsのdefault port番号は何

Short Text
Explanation: Correct: 443
