References (URLs)

  • RFC 6797: HTTP Strict Transport Security

Goal: directiveの意味と, first visit問題を説明できるようにする.

Q1: browserへHSTS policyを伝えるheaderはどれ

Multiple Choice
Explanation: Correct (B): Strict-Transport-SecurityをHTTPSで返し, browserがpolicyをcacheする

Q2: HSTS directiveとして一般的なものはどれ (複数選択)

Multi-Select
Explanation: Correct (A,B,C): max-ageは必須, includeSubDomainsとpreloadはよく使う Options: - D (incorrect): directiveではない

Q3: HSTSだけでは完全に守れない状況として典型なのはどれ

Multiple Choice
Explanation: Terms: trust on first use Correct (C): 初回がHTTPだと, policyを学習する前にactive attackerに妨害され得る Related: preload listはこの問題を緩和しやすい

Q4: max-ageの単位は何

Multiple Choice
Explanation: Correct (A): max-ageはsecondsの整数

Q5: policyを全subdomainへ適用するdirectiveは何 (token 1つ)

Short Text
Explanation: Correct: includeSubDomains Related: subtree全体へ影響するため, 段階導入が安全

Q6: HSTSを安全に導入する時, 典型の第一歩として近いのはどれ

Multiple Choice
Explanation: Terms: lock in risk Correct (B): 誤設定があるとbrowser側でlockされ, 復旧に時間がかかる Options: - A (incorrect): 準備できていないsubdomainがあると破壊的 - B (correct): 段階導入が安全 - C (incorrect): headerはsecure connectionで返す必要がある
